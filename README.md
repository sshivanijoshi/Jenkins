That means if you're on a Windows system, you would use the percent parameter name percent format. And if you're on a Mac, Linux, or Docker system, your parameters would be accessed with a dollar sign and then the parameter name
Typically, Jenkins parameters are named in all uppercase letters. You should also note that the names are case sensitive on Mac, Linux, and Docker systems, but not on Windows systems. Regardless of the operating system, it's good practice to use your names in all uppercase and then stay consistent throughout your job to prevent errors
for choice parameters, the first choice entered will be used as the default.
